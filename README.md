# CodexBowling(Nyanko Strike Rivals)

- Codex App
- GPT-5.5 Mid
- Skills: Game Studio, Image Gen

を用いて試作したブラウザで動くボウリングゲーム。
ゲームの設計と依頼実行の検証を目的とした実験プロジェクトです。

## 操作方法

- `A` / `D`: 投球位置を左右に移動
- `←` / `→`: 投球角度を調整
- `Space` または `Start Power` ボタン: タイミング入力を進める
- 1 回目のタイミング: ボール速度を決定
- 2 回目のタイミング: カーブの強さを決定

ゲームは 5 フレーム制です。ストライク、スペア、ガーターなどの結果に応じて、中央上部の演出ウィンドウとピン奥のサイネージが変化します。

## クイックスタート

```bash
npm install
npm run dev
```

ブラウザで表示されたローカル URL を開いてください。通常は次の URL です。

```text
http://127.0.0.1:5173/
```

本番ビルドを確認する場合:

```bash
npm run build
npm run preview
```

LAN内向けに0.0.0.0でVite起動する
```bash
npm run dev:lan
```
や
```bash
npm run preview:lan
```
もあります。

## GitHub Pages

`vite.config.ts` の `base` は `./` にしているため、GitHub Pages のサブパス配信でもアセットを相対パスで解決できます。

公開用の静的ファイルは次のコマンドで `dist/` に生成されます。

```bash
npm run build
```

## 使っている技術

- TypeScript: ゲームロジックと型定義
- Vite: 開発サーバーと本番ビルド
- Three.js: 3D レーン、ボール、ピン、カメラ、ライティング
- Rapier 3D: ボールとピンの物理挙動
- DOM / CSS: HUD、スコア、操作パネル、演出ウィンドウ
- 生成画像アセット: キャラクター立ち絵、スプライト、勝敗一枚絵

## 制作時の依頼概要

このゲームは、ユーザーからの段階的な依頼をもとに制作しました。

- 参考三面図のキャラクターをもとに、ボウリングウェアを着たヒロイン Mao を作る
- ヒロイン Mao とライバル Rinka が 5 フレーム制で対戦するブラウザゲームにする
- キャラクターは立ち絵、ピクセルアート風の投球アニメーション、勝敗一枚絵で表現する
- ボウリング部分は実際のボール、レーン、ピンの規格に近いサイズで、3D 物理挙動として表現する
- カメラはプレイヤーの目線に近い低めの位置に置き、投球中だけ追従カメラへ切り替える
- 左右は壁ではなくガーター溝にし、ピン奥にはボール回収用の逃げを用意して結果へ影響しにくくする
- 速度とカーブの強さをタイミング入力で決めるゲーム性を入れる
- 投球結果に応じてセリフ、中央上部の演出ウィンドウ、ピン奥のサイネージを変化させる
- ストライク、スペア、ガーターなどの結果が分かりやすいように、サイネージと画面エフェクトを強化する
- 2 投目では倒れたピンを取り除き、残っているピンだけで正しく次の投球を行えるようにする
- プレイヤーが 45 点以上を出すと勝ちやすい程度に、ライバル Rinka の強さを調整する
- リザルト画面には勝敗一枚絵とスコアボードを表示し、試合内容を振り返れるようにする
- モバイルの縦画面、横画面、正方形に近い画面でも UI が重なりにくいレイアウトにする
- LAN 内の実機ブラウザや GitHub Pages で確認、公開しやすい構成にする
